RetroArch iOS

This is a port of RetroArch for Apple iOS-based devices, such as iPod Touch, iPhone, iPad and Apple TV. RetroArch iOS was first released to the public on 28 May 2013 [version 0.9.9]. It is currently at [version 0.9.9.2].

What do I need to use it on my iOS device?

In order to use RetroArch iOS, you must meet certain requirements.

  • You have a non-jailbroken iOS device, and you are a registered Apple iOS developer ($99 per year) that knows how to compile RetroArch (and the cores you want) yourself. To see how to do this, go here.
  • You have a jailbroken iOS device, and are able to install RetroArch iOS from Cydia Store.

Where to get it?

For jailbroken users

For jailbroken users, RetroArch iOS is available on two repositories - one of them being a default Cydia repository (ZodTTD & MacCiti- http://cydia.zodttd.com/repo/cydia).

We also have our own Cydia repository - add http://themaister.net/cydia to your Cydia Sources. whenever we release new versions, it usually gets on this repo first. So if you can't wait until ZodTTD publishes our latest releases, you can install this repository instead.

For non-jailbroken users

iTunes Store has similar restrictions as Blackberry World so we can't publish RetroArch iOS on there. Therefore, there is no official way to obtain RetroArch iOS on non-jailbroken devices. Loopholes may exist, but Apple likely won't approve of such methods.

Notes

For non-jailbroken users

Apps run on non-jailbroken device have certain restrictions imposed on them. For example, an app is not allowed to generate machine code at runtime.

Because of this reason, emulator cores that are dependent on dynamic recompilation CPU cores will not be able to run.

These include:

  • PCSX ReARMed
  • Desmume

Nothing can be done about this - being able to run cores like these are the perks of being on a jailbroken device.
